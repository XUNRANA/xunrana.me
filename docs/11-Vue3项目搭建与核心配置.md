## 1. 技术选型

### 1.1 前端技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Vue 3 | 3.x | 渐进式前端框架，Composition API |
| TypeScript | 5.x | 类型安全，减少运行时错误 |
| Vite | 5.x | 极速开发服务器 + 构建工具 |
| Vue Router | 4.x | 官方路由，支持 History 模式 |
| Pinia | 2.x | Vue 3 官方状态管理，替代 Vuex |
| Axios | 1.x | HTTP 客户端，拦截器机制 |
| Element Plus | 2.x | Vue 3 组件库，后台管理系统标配 |
| md-editor-v3 | 4.x | Markdown 编辑器 + 预览组件 |

### 1.2 为什么选 Vite 而不是 Webpack

- **冷启动速度**：Vite 基于 ESM（ES Modules），不需要打包就能启动开发服务器；Webpack 需要把整个依赖图打包后才能启动
- **HMR 热更新**：Vite 的 HMR 是文件级别的，改一个文件只重新请求这一个；Webpack 需要重新构建受影响的整个模块链
- **构建速度**：生产构建用 Rollup，比 Webpack 更快
- **配置简单**：零配置就能用 TypeScript + JSX，不需要 `ts-loader`、`babel-loader` 等

**面试中怎么说**：Vite 通过 ESM + esbuild 预构建实现了亚秒级冷启动。开发时浏览器直接请求 ESM 模块，Vite 只做拦截和转换；生产构建用 Rollup 打包。这个架构让 HMR 和冷启动都比 Webpack 快一个数量级。

## 2. 项目初始化

### 2.1 创建项目

```bash
npm create vite@latest frontend -- --template vue-ts
cd frontend
npm install
```

这个命令会生成一个标准的 Vue 3 + TypeScript 项目骨架，包含：
- `vite.config.ts` — Vite 配置
- `tsconfig.json` / `tsconfig.app.json` — TypeScript 配置

### 2.2 安装业务依赖

```bash
# 核心依赖
npm install vue-router@4 pinia axios element-plus @element-plus/icons-vue md-editor-v3

# 开发依赖
```

- `vue-router@4` — Vue 3 路由

## 3. Vite 配置

### 3.1 `vite.config.ts` 完整配置

```typescript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { resolve } from 'path'

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': resolve(__dirname, 'src'),  // 路径别名
    },
  },
  server: {
    port: 5173,
    proxy: {
      '/api': {
        target: 'http://localhost:8080',  // 后端地址
        changeOrigin: true,
      },
    },
  },
})
```

### 3.2 路径别名 `@/`

`@` 指向 `src/` 目录。这样写 import 时不用算相对路径层数：

```typescript
// 不好：import request from '../../../utils/request'
// 好：import request from '@/utils/request'
```

需要在 `tsconfig.app.json` 中同步配置：

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  }
}
```

### 3.3 开发代理

`proxy` 配置把 `/api` 开头的请求转发到后端 `http://localhost:8080`。这样前端调 `/api/v1/articles` 实际请求的是 `http://localhost:8080/api/v1/articles`，解决了开发环境的跨域问题。

**为什么需要代理**：浏览器的同源策略限制了跨域请求。开发时前端在 `localhost:5173`，后端在 `localhost:8080`，端口不同就算跨域。Vite 的 proxy 在 Node.js 层转发请求，绕过了浏览器的同源限制。

## 4. TypeScript 类型系统

### 4.1 API 响应类型

后端所有接口返回统一的 `Result<T>` 结构：

```typescript
// types/api.d.ts
export interface Result<T> {
  code: number      // 200 表示成功
  message: string   // 提示信息
  data: T           // 业务数据
}

export interface PageResult<T> {
  records: T[]      // 当前页数据
  total: number     // 总记录数
  size: number      // 每页大小
  current: number   // 当前页码
  pages: number     // 总页数
}
```

### 4.2 业务类型

每个后端 VO/DTO 在前端都有对应的 TypeScript 接口：

```typescript
// types/article.d.ts
export interface ArticleVO {
  id: number
  title: string
  slug: string
  summary: string
  coverImage: string
  categoryId: number
  categoryName: string
  viewCount: number
  status: number
  createdAt: string
  updatedAt: string
  tags: TagVO[]
}

export interface ArticleDTO {
  title: string
  slug: string
  summary: string
  content: string
  coverImage: string
  categoryId: number
  tagIds: number[]
  status: number
}
```

**设计原则**：前端类型和后端 VO/DTO 保持一一对应。后端加字段时，前端也要同步更新类型定义。这样 TypeScript 编译器能在编译期发现字段名拼错、类型不匹配等问题。

## 5. Axios 封装

### 5.1 核心设计

`utils/request.ts` 封装了 Axios 实例，主要做三件事：

1. **自动附带 Token**：请求拦截器从 localStorage 读取 JWT，加到 `Authorization` header
2. **统一错误处理**：响应拦截器检查 `Result.code`，非 200 直接 reject
3. **Token 自动续期**：收到 401 时尝试用 refreshToken 获取新 token，失败则跳转登录

```typescript
// utils/request.ts
import axios from 'axios'
import type { Result } from '@/types/api'
import { useUserStore } from '@/stores/user'
import router from '@/router'

const request = axios.create({
  baseURL: '/api',
  timeout: 10000,
})

// 请求拦截器：自动附带 Token
request.interceptors.request.use((config) => {
  const userStore = useUserStore()
  if (userStore.token) {
    config.headers.Authorization = `Bearer ${userStore.token}`
  }
  return config
})

// 响应拦截器：统一错误处理 + Token 续期
request.interceptors.response.use(
  (response) => {
    const result = response.data as Result<unknown>
    if (result.code !== 200) {
      return Promise.reject(new Error(result.message))
    }
    return response
  },
  async (error) => {
    const originalRequest = error.config
    if (error.response?.status === 401 && !originalRequest._retry) {
      originalRequest._retry = true
      const userStore = useUserStore()
      try {
        await userStore.refresh()
        originalRequest.headers.Authorization = `Bearer ${userStore.token}`
        return request(originalRequest)  // 重试原请求
      } catch {
        userStore.logout()
        router.push('/admin/login')
        return Promise.reject(error)
      }
    }
    return Promise.reject(error)
  }
)

export default request
```

### 5.2 Token 续期流程

```
请求 → 401 错误
  → 用 refreshToken 请求 /v1/auth/refresh
    → 成功：更新 token，重试原请求
    → 失败：清除 token，跳转登录页
```

这个机制保证了用户在 token 过期时不会被强制登出，而是无缝续期。refreshToken 有效期 7 天，accessToken 有效期 30 分钟。

### 5.3 API 模块化

每个业务模块一个文件，职责清晰：

```typescript
// api/article.ts
import request from '@/utils/request'
import type { Result, PageResult } from '@/types/api'
import type { ArticleVO, ArticleDTO, ArticleQueryDTO } from '@/types/article'

export function getArticles(params: ArticleQueryDTO) {
  return request.get<Result<PageResult<ArticleVO>>>('/v1/articles', { params })
}

export function createArticle(data: ArticleDTO) {
  return request.post<Result<void>>('/v1/admin/articles', data)
}
```

## 6. Pinia 状态管理

### 6.1 为什么用 Pinia 而不是 Vuex

- **更简洁**：没有 mutations，直接修改 state
- **TypeScript 友好**：类型推导比 Vuex 好得多
- **Composition API 风格**：用 `setup` 函数定义 store，和组件写法一致
- **Vue 3 官方推荐**：Pinia 已经是 Vue 官方状态管理方案

### 6.2 User Store

```typescript
// stores/user.ts
import { defineStore } from 'pinia'
import { ref } from 'vue'
import type { UserVO } from '@/types/auth'
import * as authApi from '@/api/auth'

export const useUserStore = defineStore('user', () => {
  const token = ref(localStorage.getItem('token') || '')
  const refreshTokenValue = ref(localStorage.getItem('refreshToken') || '')
  const userInfo = ref<UserVO | null>(null)

  async function login(username: string, password: string) {
    const { data } = await authApi.login({ username, password })
    token.value = data.data.accessToken
    refreshTokenValue.value = data.data.refreshToken
    localStorage.setItem('token', data.data.accessToken)
    localStorage.setItem('refreshToken', data.data.refreshToken)
  }

  async function refresh() {
    const { data } = await authApi.refreshToken(refreshTokenValue.value)
    token.value = data.data.accessToken
    refreshTokenValue.value = data.data.refreshToken
    localStorage.setItem('token', data.data.accessToken)
    localStorage.setItem('refreshToken', data.data.refreshToken)
  }

  function logout() {
    token.value = ''
    refreshTokenValue.value = ''
    userInfo.value = null
    localStorage.removeItem('token')
    localStorage.removeItem('refreshToken')
  }

  return { token, refreshToken: refreshTokenValue, userInfo, login, refresh, logout }
})
```

**Token 持久化**：Token 存在 localStorage 中，页面刷新后不会丢失。Pinia 的 `ref` 负责响应式，localStorage 负责持久化。

## 7. Vue Router 路由设计

### 7.1 路由结构

```typescript
// router/index.ts
const routes = [
  {
    path: '/',
    component: () => import('@/layouts/BlogLayout.vue'),
    children: [
      { path: '', name: 'Home', component: () => import('@/views/blog/HomeView.vue') },
      { path: 'article/:slug', name: 'ArticleDetail', component: () => import('@/views/blog/ArticleDetailView.vue') },
      { path: 'category', name: 'Category', component: () => import('@/views/blog/CategoryView.vue') },
      { path: 'tag', name: 'Tag', component: () => import('@/views/blog/TagView.vue') },
      { path: 'archives', name: 'Archives', component: () => import('@/views/blog/ArchivesView.vue') },
    ],
  },
  {
    path: '/admin/login',
    name: 'AdminLogin',
    component: () => import('@/views/admin/LoginView.vue'),
  },
  {
    path: '/admin',
    component: () => import('@/layouts/AdminLayout.vue'),
    meta: { requiresAuth: true },
    children: [
      { path: 'dashboard', name: 'Dashboard', component: () => import('@/views/admin/DashboardView.vue') },
      { path: 'articles', name: 'ArticleManage', component: () => import('@/views/admin/ArticleManageView.vue') },
      // ... 其他管理页面
    ],
  },
]
```

### 7.2 路由守卫

```typescript
router.beforeEach((to) => {
  if (to.meta.requiresAuth) {
    const userStore = useUserStore()
    if (!userStore.token) {
      return '/admin/login'
    }
  }
})
```

访问 `/admin` 下的任何页面时，如果没有 token，自动跳转到登录页。

### 7.3 懒加载

所有路由组件都用 `() => import(...)` 动态导入。Vite 会自动进行代码分割，每个页面单独一个 chunk，首屏只加载当前页面的代码。

## 8. Element Plus 集成

### 8.1 全量引入

```typescript
// main.ts
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'

app.use(ElementPlus)
```

全量引入最简单，但打包体积较大（约 800KB gzip 后约 200KB）。如果后续需要优化，可以改为按需引入。

### 8.2 按需引入（可选优化）

安装 `unplugin-vue-components` 和 `unplugin-auto-import`：

```bash
npm install -D unplugin-vue-components unplugin-auto-import
```

在 `vite.config.ts` 中配置：

```typescript
import AutoImport from 'unplugin-auto-import/vite'
import Components from 'unplugin-vue-components/vite'
import { ElementPlusResolver } from 'unplugin-vue-components/resolvers'

export default defineConfig({
  plugins: [
    vue(),
    AutoImport({ resolvers: [ElementPlusResolver()] }),
    Components({ resolvers: [ElementPlusResolver()] }),
  ],
})
```

## 10. 面试常见问题

### Q1: 为什么用 Vite 而不是 Webpack？

Vite 基于浏览器原生 ESM，开发时不需要打包，冷启动速度极快。Webpack 需要先构建整个依赖图才能启动。Vite 的 HMR 也是文件级别的，改一个文件只重新请求这一个；Webpack 需要重新构建受影响的模块链。

### Q2: Axios 拦截器的作用？

- **请求拦截器**：自动附带 JWT Token，省去每个请求手动加 header
- **响应拦截器**：统一处理 `Result<T>` 结构，401 时自动续期 token

### Q3: Token 存 localStorage 安全吗？

localStorage 容易被 XSS 攻击读取。更安全的做法是存 httpOnly cookie，但这需要后端配合设置 cookie。本项目为了简化，用 localStorage + 前端拦截器方案。生产环境建议：
- 启用 HTTPS
- 设置 token 过期时间较短（30分钟）
- 用 refreshToken 续期

### Q4: Pinia 和 Vuex 的区别？

Pinia 是 Vue 3 官方推荐的状态管理库。相比 Vuex：
- 没有 mutations，直接修改 state
- TypeScript 类型推导更好
- 支持 Composition API 风格
- 更小的打包体积
