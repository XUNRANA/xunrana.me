## 2. 安装 JDK 17

### Windows

**方式一：手动安装（推荐）**

1. 访问 [Oracle JDK 下载页](https://www.oracle.com/java/technologies/javase/jdk17-archive-downloads.html) 或 [Adoptium](https://adoptium.net/temurin/releases/?version=17)
2. 下载 Windows x64 安装包 (.msi 或 .zip)
3. 安装后配置环境变量：

```bash
# 添加系统环境变量
JAVA_HOME = C:\Program Files\Java\jdk-17
# 在 Path 中添加
%JAVA_HOME%\bin
```

4. 验证安装：

```bash
java -version
# 输出: java version "17.0.x" ...

javac -version
# 输出: javac 17.0.x
```

**方式二：使用 scoop（Windows 包管理器）**

```powershell
# 安装 scoop（如果还没有）
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
Invoke-RestMethod -Uri https://get.scoop.sh | Invoke-Expression

# 安装 JDK 17
scoop bucket add java
scoop install temurin17-jdk
```

### macOS / Linux

```bash
# macOS (Homebrew)
brew install openjdk@17

# Ubuntu / Debian
sudo apt update
sudo apt install openjdk-17-jdk

# 验证
java -version
```

### 为什么选择 JDK 17？

- **LTS (长期支持版本)**: Oracle 和社区都会维护到 2029 年以上
- **Spring Boot 3.x 最低要求**: Spring Boot 3.0 起要求 Java 17+
- **语言新特性**: Records、Sealed Classes、Pattern Matching、Text Blocks 等
- **大厂标准**: 字节跳动、阿里巴巴等已大规模迁移到 JDK 17

## 3. 安装 Maven

### Windows

1. 下载 [Apache Maven](https://maven.apache.org/download.cgi)（Binary zip archive）
2. 解压到如 `C:\maven\apache-maven-3.9.6`
3. 配置环境变量：

```bash
MAVEN_HOME = C:\maven\apache-maven-3.9.6
# 在 Path 中添加
%MAVEN_HOME%\bin
```

4. 验证：

```bash
mvn -version
# 输出: Apache Maven 3.9.x ...
```

### 配置国内镜像（加速依赖下载）

编辑 Maven 配置文件 `~/.m2/settings.xml`（Windows 在 `C:\Users\你的用户名\.m2\settings.xml`）：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<settings>
    <mirrors>
        <mirror>
            <id>aliyun</id>
            <name>阿里云公共仓库</name>
            <url>https://maven.aliyun.com/repository/public</url>
            <mirrorOf>central</mirrorOf>
        </mirror>
    </mirrors>
</settings>
```

> **为什么配置镜像？** Maven 默认从 Maven Central（国外服务器）下载依赖，速度很慢。配置阿里云镜像后，从国内服务器下载，速度提升 10 倍以上。

## 4. 安装 MySQL 8.0

### 方式一：Docker 安装（强烈推荐）

```bash
# 拉取并启动 MySQL 8.0
docker run -d \
  --name blog-mysql \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=root123 \
  -v mysql-data:/var/lib/mysql \
  mysql:8.0 \
  --character-set-server=utf8mb4 \
  --collation-server=utf8mb4_unicode_ci

# 验证是否启动成功
docker ps | grep blog-mysql

# 连接测试
docker exec -it blog-mysql mysql -uroot -proot123
```

**参数说明：**
- `-d` : 后台运行
- `--name blog-mysql` : 容器名称
- `-p 3306:3306` : 端口映射（宿主机:容器）
- `-e MYSQL_ROOT_PASSWORD=root123` : 设置 root 密码
- `-v mysql-data:/var/lib/mysql` : 数据持久化（容器删了数据不丢）
- `--character-set-server=utf8mb4` : 支持中文和 emoji

### 方式二：本地安装

1. 下载 [MySQL Installer](https://dev.mysql.com/downloads/installer/)
2. 选择 "Developer Default" 安装
3. 设置 root 密码为 `root123`（与项目配置一致）
4. 确保端口 3306 未被占用

### 常用 MySQL Docker 命令

```bash
# 启动 / 停止 / 重启
docker start blog-mysql
docker stop blog-mysql
docker restart blog-mysql

# 查看日志
docker logs blog-mysql

# 进入 MySQL 命令行
docker exec -it blog-mysql mysql -uroot -proot123

# 删除容器（数据保留在 volume 中）
docker rm -f blog-mysql
```

## 5. 安装 Redis 7

### 方式一：Docker 安装（强烈推荐）

```bash
# 拉取并启动 Redis 7
docker run -d \
  --name blog-redis \
  -p 6379:6379 \
  redis:7-alpine

# 验证
docker exec -it blog-redis redis-cli ping
# 输出: PONG
```

### 方式二：Windows 本地安装

Redis 官方不直接支持 Windows，可以使用以下方案：

- [Memurai](https://www.memurai.com/)（Redis 的 Windows 移植版）
- WSL2 中安装 Linux 版 Redis

### 常用 Redis 命令

```bash
# 进入 Redis CLI
docker exec -it blog-redis redis-cli

# 常用操作
127.0.0.1:6379> SET test "hello"    # 设置键值
127.0.0.1:6379> GET test            # 获取值
127.0.0.1:6379> DEL test            # 删除键
127.0.0.1:6379> KEYS blog:*         # 查看所有 blog 前缀的键
127.0.0.1:6379> TTL blog:token:blacklist:xxx  # 查看键的剩余过期时间
127.0.0.1:6379> FLUSHDB             # 清空当前数据库（开发用）
```

打开 `backend/src/main/resources/application-dev.yml`，确认以下配置与你的环境一致：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/xunrana_blog?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true

    password: root123        # ← 改成你的 MySQL 密码

  data:
    redis:
      host: localhost
      port: 6379             # ← 确认 Redis 端口
      database: 0
```

> **注意**: 如果你的 MySQL 密码不是 `root123`，需要修改 `password` 字段。其他配置一般不需要改。

## 11. 使用 Postman / curl 测试

### curl 命令示例

```bash
# 1. 登录
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'

# 保存返回的 accessToken（以下用 $TOKEN 代替）

# 2. 获取文章列表（公开接口，不需要 Token）
curl http://localhost:8080/api/v1/articles

# 3. 创建文章（需要 Admin Token）
curl -X POST http://localhost:8080/api/v1/admin/articles \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "title": "我的第一篇博客",
    "content": "# Hello World\n\n这是我的第一篇博客文章。",
    "summary": "这是一篇测试文章",
    "categoryId": 1,
    "status": 1,
    "tagIds": [1, 2]
  }'

# 4. 获取分类列表
curl http://localhost:8080/api/v1/categories

# 5. 获取标签列表
curl http://localhost:8080/api/v1/tags

# 6. 查看用户信息（需要认证）
curl http://localhost:8080/api/v1/auth/info \
  -H "Authorization: Bearer $TOKEN"

# 7. 登出
curl -X POST http://localhost:8080/api/v1/auth/logout \
  -H "Authorization: Bearer $TOKEN"
```

### Postman 使用技巧

1. 创建一个 Collection "Xunrana Blog"
2. 设置 Collection 变量：`baseUrl = http://localhost:8080/api`
3. 登录接口的 Tests 脚本（自动保存 Token）：

```javascript
if (pm.response.code === 200) {
    var jsonData = pm.response.json();
    pm.collectionVariables.set("accessToken", jsonData.data.accessToken);
}
```

4. 其他接口在 Authorization 标签页选择 "Bearer Token"，填入 `{{accessToken}}`

## 12. IDEA 开发配置

### 推荐插件

| 插件 | 用途 |
|------|------|
| Lombok | 支持 @Data, @Slf4j 等注解 |
| MyBatisX | MyBatis Mapper 跳转、代码生成 |

### Lombok 注解处理器配置

如果 IDEA 报 `@Data` 等注解找不到 getter/setter：

1. `File` → `Settings` → `Build, Execution, Deployment` → `Compiler` → `Annotation Processors`
2. 勾选 "Enable annotation processing"
3. 重启 IDEA

### 热重载配置

开发时修改代码不想每次重启，可以使用 Spring Boot DevTools：

1. pom.xml 中已包含 `spring-boot-devtools`（可选添加）
2. IDEA 中: `File` → `Settings` → `Build, Execution, Deployment` → `Compiler`
3. 勾选 "Build project automatically"
4. `Ctrl + Shift + A` 搜索 "Registry"，勾选 `compiler.automake.allow.when.app.running`

### Debug 断点调试

1. 在代码行号左侧点击设置断点（红点）
2. 用 Debug 模式启动（绿色虫子图标）
3. 发送请求时会停在断点处
4. 可以查看变量值、单步执行、评估表达式

**推荐断点位置（学习用）：**
- `JwtAuthFilter.doFilterInternal()` — 观察 Token 认证流程
- `AuthServiceImpl.login()` — 观察登录处理流程
- `GlobalExceptionHandler` 的各个方法 — 观察异常处理流程
- `ArticleServiceImpl.createArticle()` — 观察文章创建流程

## 13. 常见问题排查

### Q1: 端口 3306 被占用

```bash
# Windows 查看端口占用
netstat -aon | findstr :3306

# 杀掉占用进程
taskkill /PID <进程ID> /F

# 或者修改 application-dev.yml 中的 MySQL 端口
```

### Q2: MySQL 连接被拒绝 (Connection refused)

可能原因：
1. MySQL 没有启动 → `docker start blog-mysql`
2. 密码错误 → 检查 `application-dev.yml` 中的 password
3. 数据库不存在 → 运行 `deploy/mysql/init.sql`

```bash
# 测试 MySQL 连接
docker exec -it blog-mysql mysql -uroot -proot123 -e "SHOW DATABASES;"
```

### Q3: Redis 连接失败

```bash
# 检查 Redis 是否运行
docker ps | grep blog-redis

# 如果没有运行
docker start blog-redis

# 测试连接
docker exec -it blog-redis redis-cli ping
# 应该返回 PONG
```

### Q4: Maven 依赖下载失败

```bash
# 清理本地仓库缓存后重试
mvn dependency:purge-local-repository
mvn clean install

# 如果是网络问题，确认配置了阿里云镜像（见第3节）

# 强制更新快照版本
mvn clean install -U
```

### Q5: IDEA 无法识别 Lombok 注解

1. 确认安装了 Lombok 插件（`Settings` → `Plugins` → 搜索 "Lombok"）
2. 确认开启了注解处理器（见第12节）
3. 清理 IDEA 缓存：`File` → `Invalidate Caches / Restart`

### Q6: 启动时报 `Table 'xunrana_blog.xxx' doesn't exist`

数据库没有初始化，运行：
```bash
docker exec -i blog-mysql mysql -uroot -proot123 < deploy/mysql/init.sql
```

### Q7: JWT Token 认证失败（401）

1. 检查 Token 是否过期（默认 30 分钟）
2. 检查 Authorization 头格式：`Bearer eyJhbG...`（注意有空格）
3. 检查是否使用了 accessToken 而非 refreshToken
4. 查看后端日志中的错误信息

### Q8: 管理员接口返回 403 Forbidden

1. 确认使用的是 admin 账户的 Token
2. 验证 Token 中的 role 字段为 1：
```bash
# 解码 JWT 的 payload（中间部分）
echo "JWT_PAYLOAD_部分" | base64 -d
# 应该看到 "role": 1
```

### Q9: Windows 上 curl 命令格式问题

Windows CMD 中 JSON 需要用双引号转义：
```cmd
curl -X POST http://localhost:8080/api/v1/auth/login ^
  -H "Content-Type: application/json" ^
  -d "{\"username\":\"admin\",\"password\":\"admin123\"}"
```

推荐使用 Git Bash 或 PowerShell，可以直接用单引号。

## 面试常见问题

### Q: 你的项目本地开发环境是怎么搭的？

> "项目使用 Docker 运行 MySQL 和 Redis 等中间件，通过 docker run 命令快速启动，数据通过 volume 持久化。后端用 IDEA 启动 Spring Boot 应用，配置了 dev/prod 多环境 Profile，敏感信息在生产环境通过环境变量注入。API 测试用 SpringDoc 自动生成的 Swagger UI 文档。"

### Q: 为什么用 Docker 而不是直接安装 MySQL/Redis？

> "主要有三个原因：一是**环境一致性**，Docker 容器保证开发/测试/生产环境完全一致，避免 '在我电脑上能跑' 的问题；二是**便捷性**，一条命令就能启动带配置的服务，不需要手动安装和配置；三是**隔离性**，不同项目可以用不同版本的 MySQL，互不影响，也不会污染宿主机环境。"

### Q: Spring Boot 的多环境配置是怎么实现的？

> "Spring Boot 通过 Profile 机制实现多环境配置。`application.yml` 是主配置文件，指定激活的 Profile（如 `spring.profiles.active: dev`）。`application-dev.yml` 和 `application-prod.yml` 分别存放开发和生产环境的配置。Spring Boot 启动时会先加载主配置，再加载对应 Profile 的配置，Profile 配置会覆盖主配置中的同名项。生产环境的敏感信息（数据库密码等）通过 `${ENV_VAR}` 占位符从环境变量读取，不写死在配置文件中。"
